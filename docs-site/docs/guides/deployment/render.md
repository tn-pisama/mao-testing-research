# Render Deployment

PISAMA includes a `render.yaml` blueprint for one-click deployment on [Render](https://render.com).

## Setup

1. Push your repository to GitHub
2. Go to [Render Dashboard](https://dashboard.render.com) > **New** > **Blueprint**
3. Connect your repository and select `render.yaml`
4. Render creates:
   - **mao-api** -- Python web service (backend)
   - **mao-db** -- PostgreSQL database (free tier)

`JWT_SECRET` and `DATABASE_URL` are auto-generated by the blueprint.

## Environment Variables

Set these in the Render dashboard under your service's Environment tab:

| Variable | Value |
|---|---|
| `REDIS_URL` | Redis connection string (use Render Redis or external) |
| `CORS_ORIGINS` | Your Vercel frontend URL |
| `GOOGLE_CLIENT_ID` | Google OAuth client ID |
| `GOOGLE_CLIENT_SECRET` | Google OAuth client secret |

## Frontend on Vercel

Deploy the frontend to Vercel:

```bash
cd frontend
vercel --prod
```

Set environment variables in Vercel:

| Variable | Value |
|---|---|
| `NEXT_PUBLIC_API_URL` | `https://mao-api.onrender.com/api/v1` |
| `NEXTAUTH_SECRET` | Generate with `openssl rand -base64 32` |
| `NEXTAUTH_URL` | `https://your-app.vercel.app` |
| `GOOGLE_CLIENT_ID` | Your Google OAuth client ID |
| `GOOGLE_CLIENT_SECRET` | Your Google OAuth client secret |

## Redis Options

Redis is used for rate limiting and caching. Options for Render:

| Provider | Pricing | Notes |
|---|---|---|
| Render Redis | $7/mo | Managed, easiest setup |
| Upstash | Free tier | Serverless Redis |
| Redis Cloud | Free tier | Managed Redis |

If Redis is unavailable, the app falls back to in-memory rate limiting (not suitable for multi-instance deployments).

## Database Alternatives

The Render blueprint uses Render's managed PostgreSQL. You can also use:

- **Supabase** -- pgvector pre-installed
- **Neon** -- pgvector available, serverless
- **AWS RDS** -- Enable pgvector extension manually

All require PostgreSQL 16+ with the pgvector extension.

## CI/CD

The repository includes GitHub Actions workflows:

- **`.github/workflows/ci.yml`** -- Runs on PRs: smoke tests, backend tests, calibration gate, frontend build, lint
- **`.github/workflows/deploy.yml`** -- Auto-deploys on push to main via Render/Vercel
