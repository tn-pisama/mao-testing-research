## 2. About Section (2600 chars max)

```
I build infrastructure for multi-agent AI systems.

**The Problem I'm Solving**

Last year, I was building multi-agent systems and kept running into failures that existing tracing tools couldn't diagnose. LangSmith would show me traces, but not patterns. I needed to know "why did this agent loop?" not just "what steps happened?"

I discovered the MAST (Multi-Agent System Testing) taxonomy—academic research defining 17 failure modes specific to multi-agent systems. Infinite loops, state corruption, persona drift, coordination breakdowns—failures that slip through traditional testing. No commercial tools addressed this. That's a gap worth filling.

**What I Built**

PISAMA: A testing platform with 17 specialized failure detectors for multi-agent AI systems.

Over 6 months, I:
• Designed and shipped a full-stack platform: FastAPI backend, Next.js frontend, Python SDK
• Built 17 detection algorithms (9 with perfect F1 scores in benchmarks)
• Created adapters for LangGraph, CrewAI, AutoGen, and n8n
• Implemented tiered detection architecture (cost optimization: $5/trace → $0.05/trace)
• Integrated OpenTelemetry with gen_ai.* semantic conventions
• Wrote 60+ test files and 11 E2E Playwright tests
• Preparing for Product Hunt launch Q1 2026

**Technical Stack**
Backend: FastAPI, PostgreSQL, pgvector, OTEL, Alembic
Frontend: Next.js 16, React 18, TailwindCSS, Google OAuth
SDK: Python with framework adapters
Infrastructure: Docker, Fly.io, Vercel, GitHub Actions

**The Impact**

PISAMA helps teams building production multi-agent systems catch failures before users do. We detect the failures that generic observability tools miss—the coordination breakdowns, infinite loops, and state corruption that cost thousands in API bills and countless hours debugging.

**What's Next**

• Launching on Product Hunt Q1 2026
• Growing design partner program
• Adding self-healing capabilities
• Expanding framework support

**Open To**
• Speaking engagements on multi-agent testing
• Technical advisory roles
• Consulting on agent infrastructure
• Connecting with teams building multi-agent systems

Let's connect if you're working on multi-agent AI, LLM operations, or developer tools.
```
